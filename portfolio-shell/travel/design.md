# Design Guide — travel project design guide
> 이 문서는 이 프로젝트의 디자인 기준이다.
> 모든 화면과 컴포넌트는 이 문서를 먼저 읽고 따른다.
> 규칙과 어긋나면 만들기 전에 멈추고 나에게 물어본다.

> **현재 확정 화면 스펙:** `design_handoff_travel_web/README.md`(+ `Travel 웹화면.dc.html`)가
> 웹 원페이지의 최신 하이파이 기준이다. 이 프로젝트의 `index.html`/`css`/`js`는 그 핸드오프를
> 재현한 것. 아래 토큰(색·타이포·간격)은 핸드오프와 일치하며, 화면 구조/인터랙션의 세부는
> 핸드오프 README를 우선한다.

## 1. Overview (브랜드 한 줄·무드)
- 브랜드명: travel
- 한 문장: 여행하기 좋은 나라 (환율 기반 추천 서비스)
- 무드 키워드: 절제된 / 타입 중심 / 시원시원한 포인트

## 2. Colors (컬러)
- Primary (포인트): #0E4AEB
- Secondary (서브 포인트): #06B6D4
- Ink (제목·본문): #0F172A
- Muted (보조 텍스트): #64748B
- Surface (카드·띠): #F5F8FF
- Background (배경): #FFFFFF
- Danger (하락/불리): #EF4444
- Success (상승/유리): Primary(#0E4AEB) 재사용
- 규칙: 색은 위계에만. 포인트는 아이콘·브랜드명·버튼에만, 카드는 중립면.

## 3. Typography (타이포그래피)
- 폰트: Pretendard (전부 이 하나만)
- H1 64px / 800 · H2 32px / 800 · H3 18px / 700 · 본문 16px / Regular
- 라벨 14px / 700 (Primary #0E4AEB) · 행간 본문 1.6
- 규칙: 폰트는 하나. 위계는 크기·굵기로만.

## 4. Layout (간격·여백)
- 기준 단위: 8px 그리드 (8 / 16 / 24 / 32 / 48 / 64 / 96)
- 콘텐츠 최대 너비 1080px · 섹션 패딩 96px · 카드 간격 24px · 좌우 여백 24px
- 규칙: 위 단위 밖 어중간한 값 금지.

## 5. Elevation & Depth (그림자·깊이)
- 기본은 그림자 없음 — 면과 여백으로 구분한다.
- 호버 시에만 얕게: 0 16px 38px rgba(20,20,30,.08) + 살짝 떠오르기.
- 규칙: 라이트그레이 구분선으로 영역을 나누지 않는다.

## 6. Shapes (모서리·형태)
- 모서리: 카드 22px / CTA 패널 28px / 버튼 pill(완전 둥글게)
- 아이콘: 라인(블랙 스트로크) + Primary 컬러 도트
- 규칙: 둥글기 값은 위 세 가지로만 통일.

## 7. Components (컴포넌트)
- Primary 버튼: 배경 #0E4AEB, 흰 글자, pill, 높이 50px
- Ghost 버튼: 흰 배경, 1px #E4E4E8 보더, pill
- 카드: 배경 #F5F8FF, 모서리 22px, 호버 시 살짝 떠오름
- 통계: 큰 숫자 44px / 800 + 보조 라벨
- 국가 카드: 국기 아이콘 + 국가명 + 변동률(%, 화살표) + 클릭 시 상세 이동
  - 변동률 유리: Primary(#0E4AEB) 색, ▼ 아이콘 (환율 하락=원화 강세, 실제 환율 방향으로 표기)
  - 변동률 불리: Danger(#EF4444) 색, ▲ 아이콘 (환율 상승=원화 약세)
  - 화살표는 "유리/불리"가 아니라 "실제 환율 등락 방향"을 나타낸다. 숫자는 절댓값(크기)으로 표기.
- 피처 히어로 카드(1위 강조): 메인 목록 맨 위, 1위를 크게 강조. 큰 국기 + 라벨("지금 가장 유리한 여행지") + 큰 국가명 + 큰 변동률(카운트업) + 체감 문구("10만원어치 쓰면 약 N원 이득"). 강조는 그림자 대신 Primary 링(inset)으로.
- 리더보드 행(2위~): 한 줄 행 = 순위 · 국기 · 국가명(+환율) · 변동률. 얇은 hairline 구분선으로 나열, 호버 시 옅은 면 강조. (박스 카드 대신 밀도 있는 리스트)

## 8. Do's and Don'ts (지킬 것·하지 말 것)
- 폰트 2개 이상 혼용 금지
- 카드 알록달록 채색 금지 (중립면 + 아이콘만 컬러)
- 그라데이션·과한 그림자·라이트그레이 선 금지
- 박스 좌측 컬러 스트라이프 금지
- 모션은 페이드업·카운트업, 그리고 스크롤 진입 시 페이드업(IntersectionObserver) 정도로 절제

## 9. Responsive (반응형)
- 브레이크포인트: 1080px(데스크톱) / 768px(태블릿) / 480px(모바일)
- 모바일: 섹션 패딩 24px, 카드 간격 16px, 카드 그리드 1열

## 10. States (상태)
- 로딩: 스켈레톤 UI (카드 형태 유지한 채 회색 placeholder) — 스피너 지양
- 에러: 카드 영역에 "불러오지 못했어요" + 재시도 버튼
- 포커스(키보드 접근성): Primary 색 2px 아웃라인
- 빈 상태(데이터 없음): Muted 색 텍스트로 안내 문구만, 별도 일러스트 없음

## 11. Icons & Images (아이콘·이미지)
- 국기 표시: 실제 국기 이미지(flagcdn SVG) 사용. 모서리 6px 둥글게.
  (Windows가 국기 이모지를 지원하지 않아 이모지는 "TH" 같은 코드로 깨짐 → 이미지로 대체)
- 그 외 아이콘: 라인 스트로크 아이콘 (섹션 6 규칙과 동일), 별도 아이콘 폰트/라이브러리 없이 인라인 SVG로 직접 관리
- 이미지: 이번 프로젝트는 데이터 중심이라 배경 이미지·일러스트 없이 타이포와 색으로만 구성
- 규칙: 아이콘 색은 라인(블랙) 또는 Primary 둘 중 하나만, 그 외 색 사용 금지
