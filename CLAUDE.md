# CLAUDE.md

## 프로젝트 개요
개인 포트폴리오 저장소. 주제/기술 스택은 추후 확정 예정.

## Git 워크플로우
- 평소 작업 중에는 커밋/푸시하지 않고 파일 변경만 진행한다.
- 사용자가 "여기까지 커밋해줘"라고 명시적으로 요청할 때만 커밋 + 푸시를 진행한다.
- 커밋 메시지는 한글로 간단하게 작성한다. (예: "헤더 컴포넌트 디자인 수정")

## 디자인 작업 규칙
- 각 프로젝트에는 자체 `design.md`가 있다. (예: `portfolio-shell/travel/design.md`)
- 해당 프로젝트에서 디자인·UI 작업을 할 때는 **매번 그 프로젝트의 design.md를 먼저 읽고** 규칙을 따른다.
- design.md 규칙과 어긋나는 부분이 있으면, 만들기 전에 멈추고 사용자에게 먼저 물어본다.
- 이 규칙은 "실행 여부 확인"의 예외다. (디자인 충돌 시에는 물어보지 않고 진행하지 않는다)

## 실행 여부 확인
- 보안/개인정보(인증키, 민감정보 노출 등)에 저촉되거나 명백히 위험한 행동이 아니면, 실행 전에 다시 물어보지 않고 바로 진행한다.
- 커밋/푸시는 위 Git 워크플로우 규칙(명시적 요청 시에만)을 그대로 따른다 — 이건 확인 절차가 아니라 애초에 자동 실행 대상이 아니기 때문.
